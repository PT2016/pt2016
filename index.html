Reguli Laborator PT

  • prezenta

    • participarea la toate laboratoarele este obligatorie
    • se accepta maxim 3 absente in situatii exceptionale (daca sunt anuntata inainte)

  • teme

    • 5 teme cu predare din 2 in 2 saptamani
    • fiecare saptamana de delay => -1p la nota finala a temei respective
    • intr-un laborator se poate preda o singura tema restanta
    • in saptamana premergatoare sesiunii se poate preda o singura tema restanta
    • exista posibilitatea obtinerii unor bonusuri anuntate pe parcursul semestrului

  • predarea temelor

    • codul sursa
    • javadoc pe clasa si pe metodele cu logica de business
    • documentatie (3000 de cuvinte, accent pe diagrame: use case, class & sequence) avand structura
      • Obiectivul temei
      • Analiza problemei, modelare, scenarii, cazuri de utilizare
      • Proiectare (diagrame UML, structuri de date, proiectare clase, interfete, relatii, packages, algoritmi, interfata utilizator)
      • Implementare si testare
      • Rezultate
      • Concluzii, ce s-a invatat din tema, dezvoltari ulterioare
      • Bibliografie
    • la predare, veti primi o intrebare intrebare legata de tema (fie pe baza codului, fie pe baza documentatiei)

  • if (labFinalGrade >= 5 && attendances >= 11) { return "pass"; } else { return "fail"; }
